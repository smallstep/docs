### macOS

Install `step` and `step-ca` together, via [Homebrew](https://brew.sh/):

```shell
brew install step
```

### Windows

#### Winget package

We publish a [Winget](https://learn.microsoft.com/en-us/windows/package-manager/winget/)
package for `Smallstep.step-ca`. To install both `step-ca.exe` and `step.exe`, run:

```
winget install Smallstep.step-ca
```

To uninstall, you can use `winget uninstall`.
Then, remove the `$HOME/.step` configuration directory.

### Scoop package

We also publish a [Scoop package](https://scoop.sh/):

```
scoop bucket add smallstep https://github.com/smallstep/scoop-bucket.git
scoop install smallstep/step
scoop install smallstep/step-ca
```

To remove it, you can use `scoop uninstall`.
Then, remove the `$HOME/.step` configuration directory.

### Linux Packages (amd64)

<Alert severity="info">
  <AlertTitle>Install both <Code>step-ca</Code> and the <Code>step</Code> CLI tool.</AlertTitle>
  <div>
    The <a href="https://smallstep.com/docs/step-cli"><Code>step</Code> CLI tool</a> is the easiest way to
    initialize, configure, and control <Code>step-ca</Code>. While <Code>step</Code> is not
    technically required to run <Code>step-ca</Code>, it is very much recommended.
  </div>

#### Debian

1. Install `step`

    Download and install the Debian package for your platform from our [latest release](https://github.com/smallstep/cli/releases/latest):
    
    ```shell
    wget https://dl.step.sm/gh-release/cli/docs-ca-install/v0.23.2/step-cli_0.23.2_amd64.deb
    sudo dpkg -i step-cli_0.23.2_amd64.deb
    ```

2. Install `step-ca`

    Download and install the Debian package for your platform from our [latest release](https://github.com/smallstep/certificates/releases/latest):

    ```shell
    wget https://dl.step.sm/gh-release/certificates/docs-ca-install/v0.23.2/step-ca_0.23.2_amd64.deb
    sudo dpkg -i step-ca_0.23.2_amd64.deb
    ```

To configure `step-ca` as a daemon, see our [systemd configuration guide][systemd-setup].

To uninstall, run `dpkg -r step-cli step-ca` and remove the configuration directory `$HOME/.step`.

#### Arch Linux

There are community-driven Arch Linux packages.

You can use [pacman](https://www.archlinux.org/pacman/) to install the packages.

```shell
pacman -S step-cli step-ca
```

The binary tarballs can be found here:
- [`step-ca`](https://archlinux.org/packages/community/x86_64/step-ca/)
- [`step-cli`](https://archlinux.org/packages/community/x86_64/step-cli/)

Big shout out to the maintainers of these packages! We appreciate you.

To uninstall, run `pacman -R step-ca step-cli` and remove the configuration directory `$HOME/.step`.

#### RedHat

1. Install `step`

    Download and install the RPM package for your platform from our [latest release](https://github.com/smallstep/cli/releases/latest):
    
    ```shell
    wget https://dl.step.sm/gh-release/cli/docs-ca-install/v0.23.2/step-cli_0.23.2_amd64.rpm
    sudo dnf install step-cli_0.23.2_amd64.rpm
    ```

2. Install `step-ca`

    Download and install the RPM package for your platform from our [latest release](https://github.com/smallstep/certificates/releases/latest):

    ```shell
    wget https://dl.step.sm/gh-release/certificates/docs-ca-install/v0.23.2/step-ca_0.23.2_amd64.rpm
    sudo dnf install step-ca_0.23.2_amd64.rpm
    ```

To configure `step-ca` as a daemon, see our [systemd configuration guide][systemd-setup].

To uninstall, run `dnf remove step-cli step-ca` and remove the configuration directory `$HOME/.step`.

There are community driven Alpine Linux Packages for `step` and `step-ca`.

To uninstall, run `apk del step-cli step-certificates` and remove the `$HOME/.step` configuration directory.

The binary tarballs can be found here:
- [`step-cli`](https://pkgs.alpinelinux.org/package/edge/community/x86/step-cli)
- [`step-certificates`](https://pkgs.alpinelinux.org/package/edge/testing/x86/step-certificates)

### Linux Binaries

We distribute pre-compiled binaries for amd64, arm64, armv6, armv7, mips, mips64, ppc64le, and 386.
See the [latest release](https://github.com/smallstep/cli/releases/latest) page.

We use [cosign](https://github.com/sigstore/cosign) for signing and verifying release artifacts.
See [Binary Signatures](#binary-signatures) for details.

Here's how to download and install the `step` and `step-ca` binaries on an amd64 machine:

1. Install `step`

    Download and install the Linux tarball from our [latest release](https://github.com/smallstep/cli/releases/latest):

    ```shell
    curl -L https://dl.step.sm/gh-release/cli/docs-ca-install/v0.23.2/step_linux_0.23.2_amd64.tar.gz \
       -o step.tar.gz
    tar -xf step.tar.gz
    sudo cp step_0.23.2/bin/step /usr/bin
    ```

2. Install `step-ca`

    Download and install the Linux tarball from our [latest release](https://github.com/smallstep/certificates/releases/latest):

    ```shell
    curl -L https://dl.step.sm/gh-release/certificates/docs-ca-install/v0.23.2/step-ca_linux_0.23.2_amd64.tar.gz \
       -o step-ca.tar.gz
    tar -xf step-ca.tar.gz
    sudo cp step-ca_0.23.2/step-ca /usr/bin
    ```

To configure `step-ca` as a daemon, see our [systemd configuration guide][systemd-setup].

[systemd-setup]: https://smallstep.com/docs/step-ca/certificate-authority-server-production#running-step-ca-as-a-daemon

#### Binary Signature Verification

We use [cosign](https://github.com/sigstore/cosign) for signing and verifying release artifacts.
To verify the signature of the `step-ca` binary:

1. [Install cosign](https://docs.sigstore.dev/cosign/installation/)
2. Download the binary signature (`.sig`) and certificate (`.pem`) files from the Assets section of our [GitHub Releases page](https://github.com/smallstep/certificates/releases/latest)
3. Run:
   
   ```
   cosign verify-blob \
     --certificate step-ca_linux_0.23.2_amd64.tar.gz.pem \
     --signature step-ca_linux_0.23.2_amd64.tar.gz.sig \
     --certificate-identity-regexp "https://github\.com/smallstep/certificates/.*" \
     --certificate-oidc-issuer https://token.actions.githubusercontent.com \
     step-ca_linux_0.23.2_amd64.tar.gz
   ```

### Kubernetes

We publish [Helm charts](https://smallstep.github.io/helm-charts/) for easy installation on kubernetes:

```shell
helm repo add smallstep https://smallstep.github.io/helm-charts/
helm repo update
helm install step-certificates smallstep/step-certificates
```

You can see all the configuration options at https://hub.helm.sh/charts/smallstep/step-certificates.
